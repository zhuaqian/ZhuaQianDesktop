# ZhuaQian Desktop 产品需求文档

更新时间：2026-07-11

## 1. 产品定位

ZhuaQian Desktop 是一个面向 Windows 的本地优先 AI 工作台。

它不是单纯聊天工具，也不是 IDE 附件，而是一个能理解文档、调用多模型、生成真实文件、管理任务状态、受权限控制执行桌面操作的轻型 Agent 产品。

## 2. 目标用户

核心用户：

- 个人办公用户
- 中小团队知识工作者
- 内容、运营、财务、行政、项目人员
- 希望在本地优先前提下使用 AI 的开发者和高级用户

典型场景：

- 读文档并总结
- 把聊天结果真实导出为 Word/PPT/Excel/TXT/MD
- 构建本地知识库并检索
- 批量整理文件
- 用截图、剪贴板、当前窗口上下文辅助工作
- 在明确授权下执行桌面侧动作

## 3. 产品目标

### 当前阶段目标

做成一个“当前就好用”的 Windows 桌面 AI 助手原型：

- 能稳定聊天
- 能读文件
- 能真落盘文件
- 能做多任务管理
- 能记录动作和产物
- 能控制风险权限

### 中期目标

从“聊天+工具”升级为“任务型 Agent 工作流”：

- Brief
- Plan
- Approve
- Execute
- Review
- Output
- Rollback

### 长期目标

成为开源的桌面 AI 工作台入口：

- 本地优先
- 插件生态
- MCP/Skills 能力
- 跨设备联动

## 4. 产品原则

1. 本地优先  
   默认在本地完成配置保存、文档处理、知识索引和日志记录。

2. 真实交付  
   模型不能只“说它做了”，桌面端必须真实执行并落盘可验证结果。

3. 权限显式  
   风险动作必须受 Power 开关和细分权限控制。

4. 任务化而非纯对话  
   每次工作应沉淀为任务、状态、动作记录和产物记录。

5. 开源可协作  
   文档、模块、测试和目录结构要适合多人协作和 AI 协作。

## 5. 核心能力范围

### 5.1 对话与任务

- 多任务左侧栏
- 任务状态分组
- Ask / Draft / Plan / Execute 模式
- 当前任务上下文保留

### 5.2 多模型接入

- Gemini
- OpenRouter
- Local
- OpenAI-compatible 扩展

### 5.3 文件理解

- 图片
- PDF
- Word
- PowerPoint
- Excel
- Markdown
- TXT
- CSV
- JSON

### 5.4 真实文件交付

- 导出最新回复为 `.txt`
- 导出最新回复为 `.md`
- 导出最新回复为 `.docx`
- 导出最新回复为 `.pptx`
- 导出最新回复为 `.xlsx`
- 导出整段聊天

### 5.5 知识库

- 文件夹索引
- chunk 和 metadata
- keyword 检索
- 向量持久化
- hybrid 检索初版

### 5.6 桌面辅助

- Screenshot OCR
- 剪贴板监控
- 当前窗口上下文
- 文件整理
- 资源监控
- 插件运行

### 5.7 安全与审计

- Power 总开关
- 细分权限
- 网络上传确认
- API Key DPAPI 保护
- 动作日志
- 产物日志
- rollback manifest

## 6. 当前版本必须满足的功能需求

### P0 必须满足

- 应用可以稳定启动
- 至少一个 provider 可正常聊天
- 文件上传后可提取内容并进入上下文
- 用户要求生成文件时，桌面端能真实落盘
- 多任务可创建、切换、保存
- 风险动作具备权限控制
- 关键动作有本地审计记录

### P1 应完善

- Outputs 成果中心可按任务查看
- Approval Card 覆盖高风险确认
- Plan 模式进入结构化审批流
- 知识库支持更稳的混合检索
- UI 减少弹窗堆叠

### P2 应演进

- 真正的 Agent 状态机
- 插件 manifest 与技能生态
- 自动化任务
- MCP 接入
- 跨设备联动

## 7. 非功能需求

### 稳定性

- 构建脚本可重复运行
- 基础测试链路保持可运行
- 失败应有可读报错，避免静默吞错

### 安全性

- 本地敏感信息默认不上传
- 用户必须知道哪些内容会进入云端
- 权限拒绝不能被悄悄绕过

### 可维护性

- 避免继续把能力堆进单文件
- 模块职责清晰
- 文档要有唯一入口

### 开源协作性

- 目录清晰
- 历史文档归档
- 核心文档少而准
- 每个执行任务应有验收标准

## 8. 当前不作为主目标的事项

- 立即跨平台重写
- 复杂营销官网
- 花哨动画优先级高于稳定性
- 隐蔽或强制性的系统级监控与自动处置
- 未定义权限模型前的大规模自动执行

## 9. 成功标准

当下版本达到以下标准，才算“当前好用版本”：

- 新用户可在 10 分钟内配置并开始工作
- 生成的文件真实存在且可打开
- 任务、动作、产物三条链能对应起来
- 关键权限路径可解释、可拒绝、可审计
- 文档足够清晰，其他 AI 能基于它直接接手开发
