## 落实修复 1（Section 二）：行数预算"口径不一致"根因收口

用户第二次抓到同一类 bug：commit 76ea300 把 maxMainLines 改成 3610（非空行口径），
但 check-architecture.ps1 用 `(Get-Content | Measure-Object -Line).Lines` 统计全行数。

**实测验证（信任但验证）**：
- `wc -l`(全行) = 3874；`grep -cve '^[[:space:]]*$'`（非空行）= 3610 —— 用户数得对。
- 但 `(Get-Content | Measure-Object -Line).Lines` 在本机 PowerShell 5.1 上**返回 3610**（≈非空行，
  非 3874）。脚本当前 PASS 是"PowerShell 版本偶然行为"的巧合，不是口径已统一。
- 根因成立：脚本机器计数与人工核对两条路径从未显式统一，换 PS 版本/编码/换行符即可能跳到 3874 立刻 FAIL。

**修复（根治，非改数字）**：
- `src/scripts/check-architecture.ps1` 两处计数（主文件 line 21 + 每文件 line 30）改为显式去空行：
  `(Get-Content -LiteralPath $x | Where-Object { $_.Trim() -ne "" } | Measure-Object).Count`。
- `docs/_line_budget.json` 加 `_countingMethod` 字段，写明"non-blank lines"，并警告勿用 `wc -l`/编辑器行数核对。
- 重跑脚本 → **Architecture checks passed.**（现确定性算 3610，不再依赖 PS 偶然行为）。
